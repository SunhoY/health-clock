# 화면기획서: S-02 프리셋 선택 화면

---

## 📌 개요

* **화면명**: 프리셋 선택
* **목적**: 저장된 루틴을 빠르게 선택하거나 새 루틴을 추가하게 한다

---

## 📂 구성 요소

* **상단 제목**: 프리셋 선택
* **프리셋 리스트**:

  * 기본 노출 프리셋: 2개
  * 심플 리스트 형태 (루틴명 + 운동 개수만 노출)
  * 각 항목 우측 액션 아이콘은 `>` 대신 햄버거(≡) 표시로 제공
  * 각 항목 탭 시 S-07 운동 진행 화면으로 이동
  * 액션 메뉴 열기 방식:
    * 햄버거 버튼 클릭
    * 또는 루틴 항목 롱클릭
  * 액션 메뉴 항목:
    * `[편집]`
    * `[삭제]`

* **빈 상태 화면** (프리셋이 없을 경우):

  * 안내 문구: 저장된 루틴이 없어요.
  * `[루틴 만들기]` 버튼 ➝ S-03 이동

* **하단 고정 버튼 (Primary, Mobile)**:

  * `[루틴 만들기]` (엄지 도달 영역 + safe area 고려)
  * 클릭 시 S-03 루틴 만들기 플로우 진입

* **모바일 UI 기준**:

  * 리스트 항목 전체를 탭 가능 영역으로 제공
  * 부가 정보(세트 상세/마지막 사용일)는 기본 화면에서 숨김
  * CTA 문구 통일: "루틴 만들기"
  * 한 화면 핵심 행동은 "프리셋 선택"으로 유지

---

## 🔁 동작

* 기존 프리셋 선택 시 ➝ 운동 바로 시작
* 햄버거 클릭 또는 롱클릭 시 ➝ 루틴 액션 메뉴 노출 (`편집/삭제`)
* `[편집]` 선택 시 ➝ 선택한 루틴의 "운동 목록 편집 단계"로 이동 (S-04 편집 모드)
* `[삭제]` 선택 시 ➝ 삭제 확인 후 삭제 API 호출(모킹) ➝ 성공 시 목록에서 제거
* 삭제 동작은 서버 연동 전 단계에서 "API 호출부를 모킹"하고, 응답 성공 시 로컬 상태에서 제거한 뒤 목록을 재조회(refetch 모킹)한 결과를 다시 렌더링한다
* `[루틴 만들기]` 클릭 시 ➝ S-03 이동
